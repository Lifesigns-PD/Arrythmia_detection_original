3. Rule Conditions — Exact Thresholds

Signal-Level Rules (derive_rule_events)

AF / Atrial Fibrillation

rr_cv = std(RR) / mean(RR)
rr_cv > 0.20
AND beats ≥ 6
AND not is_af (guard)
Fires: "AF" event (priority 90)
Also promotes background_rhythm → "AF"

Bundle Branch Block

qrs_vals = [qrs for qrs in qrs_durations_ms if qrs > 0]
mean(qrs_vals) > 120 ms
AND not is_af
Fires: "Bundle Branch Block" event (priority 70)
Independent of AV block rules — can co-fire with any block level

3rd Degree AV Block (Complete)

HR < 50 bpm
AND pr > 0 ms (P-waves detected)
AND len(per_beat_pr_ms) ≥ 3
AND std(per_beat_pr_ms) > 40 ms
AND not is_af
Fires: "3rd Degree AV Block" event (priority 80)
Sets block_fired=True → prevents lower-grade blocks firing

2nd Degree AV Block Type 2 (Mobitz II)

len(per_beat_pr_ms) ≥ 2
AND len(rr_arr) ≥ 3
AND CV(PR) = std(PR)/mean(PR) < 0.10
AND max(RR) > 1.7 × median(RR)
AND not is_af AND not block_fired
Fires: "2nd Degree AV Block Type 2" event (priority 70)
Fixed PR (no lengthening) + sudden dropped beat

2nd Degree AV Block Type 1 (Wenckebach)

len(per_beat_pr_ms) ≥ 3
AND len(rr_arr) ≥ 3
AND ≥50% of PR diffs > +10 ms (increasing trend)
AND max(RR) > 1.5 × median(RR)
AND not is_af AND not block_fired
Fires: "2nd Degree AV Block Type 1" event (priority 65)
Progressive PR lengthening + dropped beat

1st Degree AV Block

median(per_beat_pr_ms) > 200 ms
  — OR —
pr_interval (scalar) > 200 ms
AND not is_af AND not block_fired
Fires: "1st Degree AV Block" event (priority 50)
Lowest-grade block — only fires if no higher block detected

Pause

max(RR) > 2.5 s
  — equivalently —
max(RR) > 3 × median(RR)
Fires: "Pause" event (priority 85)
Independent of all other rules; can co-fire

Beat-Pattern Rules (apply_ectopy_patterns)

VT — Ventricular Tachycardia

consecutive PVC count ≥ 11 beats
(more than 10 PVCs in 10s strip)
Fires: "VT" event (priority 100)
Also promotes background_rhythm → "VT"

NSVT — Non-Sustained VT

consecutive PVC count = 4 to 10 beats
(more than 3, up to 10 in 10s strip)
Fires: "NSVT" event (priority 90)
Promotes background_rhythm → "NSVT"

Ventricular Run (Ventricular Triplet)

consecutive PVC count = exactly 3 beats
Fires: "Ventricular Run" event (priority 40)
Promotes background_rhythm → "Ventricular Run"

SVT — Supraventricular Tachycardia

consecutive PAC count ≥ 11 beats
(more than 10 PACs in 10s strip)
Fires: "SVT" event (priority 80)

PSVT — Paroxysmal SVT

consecutive PAC count = 6 to 10 beats
(more than 5, up to 10 in 10s strip)
Fires: "PSVT" event (priority 85)

Atrial Run (Atrial Triplet)

consecutive PAC count = 3 to 5 beats
Fires: "Atrial Run" event (priority 40)

PVC Bigeminy / Trigeminy / Quadrigeminy

PVC beat_indices sorted, diff analysis:
bigeminy: diffs = 1 (alt every beat)
trigeminy: diffs = 2 (every 3rd)
quadrigeminy: diffs = 3 (every 4th)
AND ≥ 3 PVC beats in window
Fires: "PVC Bigeminy/Trigeminy/Quadrigeminy" (priority 55)

PVC Couplet

Exactly 2 PVC beats adjacent
AND no consecutive run > 2
Fires: "PVC Couplet" event (priority 30)

PAC Bigeminy / Trigeminy / Quadrigeminy

Same diff-based logic as PVC variants
applied to PAC beat_indices
Fires: "PAC Bigeminy/Trigeminy/Quadrigeminy" (priority 55)

Atrial Couplet

Exactly 2 PAC beats adjacent
Fires: "Atrial Couplet" event (priority 30)

Single PVC / PAC

Ectopy model confidence ≥ 0.70
beat_label in ["PVC", "PAC"]
Fires: individual beat events (priority 10 each)
Below 0.70 → defaults to "None" class (ignored)

4. Mutual Exclusion Guards

Guard Variable Prevents Logic
AF Guard is_af BBB, all AV Block rules, Pause Checked before each block/BBB rule fires. AF's irregular RR would produce false positives in block detectors.
Block Grade Guard block_fired Lower-grade blocks co-firing 3rd Degree → sets block_fired=True → Mobitz II skipped → Wenckebach skipped → 1st Degree skipped. Only highest-grade fires.
VT Escalation run length comparison NSVT and VT co-firing apply_ectopy_patterns() replaces NSVT event with VT when run ≥5. Only one fires.
SVT Escalation run length comparison PSVT and SVT co-firing Same mechanism — PSVT replaced by SVT when PAC run ≥5.
Ectopy Threshold BEAT_ECTOPY_THRESHOLD = 0.70 Low-confidence PVC/PAC false positives xai.py: if per-beat confidence < 0.70, beat_label forced to "None". No ectopy event created.
Normal Sinus Skip label check in orchestrator + gRPC Sinus Rhythm sent as alert orchestrator: ML events not created for "Sinus Rhythm"/"Unknown". gRPC: alerts skipped for those types.

8. Data Flow — What Each Feature Enables

Feature Source Used By Rules Notes
mean_hr 60000 / mean(RR) background_rhythm, AF, 3rd Degree Primary HR from detected R-peaks
rr_intervals_ms diff(r_peaks) × 1000 / fs AF (CV), Pause, 2nd Degree Type 1 & 2 All RR-based rules depend on this list
pr_interval median(per_beat_pr_ms) via NK2 DWT 1st Degree AV Block Scalar fallback when per-beat list is short
per_beat_pr_ms NK2 DWT P-onset → R-onset per beat 2nd Degree Type 1, Type 2, 3rd Degree, 1st Degree NEW — required for Wenckebach / Mobitz II / Complete Block detection
qrs_durations_ms NK2 DWT R-onset → R-offset per beat Bundle Branch Block mean > 120ms triggers BBB rule
SDNN, RMSSD, pNN50 HRV metrics from RR intervals XAI explanations (informational) Not used in rule logic directly
beat_events xai.py ectopy model, per beat apply_ectopy_patterns() — all PVC/PAC pattern rules Each beat: {label, confidence, beat_idx, peak_sample}
sqi.is_acceptable Signal Quality Index check Artifact event creation False → SegmentState.UNRELIABLE, Artifact event fired
